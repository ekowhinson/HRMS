## 1. Introduction

### 1.1 About HRMS

HRMS (Human Resource Management System) is a comprehensive platform designed to streamline HR operations including:

- Employee information management
- Leave tracking and approvals
- Payroll processing
- Performance appraisals
- Benefits and loan management
- Comprehensive reporting

### 1.2 System Requirements

- **Browser:** Chrome, Firefox, Safari, or Edge (latest versions)
- **Screen Resolution:** Minimum 1280 x 720
- **Internet Connection:** Stable broadband connection

### 1.3 User Roles

| Role | Access Level |
|------|--------------|
| **Employee** | Self-service portal only |
| **Manager** | Team management + self-service |
| **HR Staff** | Full HR module access |
| **HR Admin** | Full access including setup |
| **System Admin** | Complete system access |

## 4. Employee Management

### 4.1 Viewing Employees

1. Navigate to **Employees** in the sidebar
2. Use filters to narrow results:
   - Department
   - Status (Active, Probation, etc.)
   - Staff Category
3. Use the search bar to find specific employees
4. Click on an employee row to view details

### 4.2 Adding a New Employee

1. Click **Add Employee** button
2. Fill in the required sections:

**Personal Information:**
- Title, First Name, Middle Name, Last Name
- Date of Birth, Gender, Marital Status
- Nationality, Ghana Card Number
- SSNIT Number, TIN Number
- Contact details (email, phone)

**Address Information:**
- Residential Address
- Postal Address
- Digital Address

**Employment Details:**
- Employee Number
- Employment Type (Permanent, Contract, etc.)
- Date of Joining
- Division, Directorate, Department
- Position, Grade
- Staff Category
- Supervisor

**Bank Details:**
- Bank Name
- Bank Branch
- Account Number

3. Click **Save** to create the employee record

### 4.3 Editing Employee Information

1. Navigate to the employee's profile
2. Click the **Edit** button
3. Update the necessary fields
4. Click **Save Changes**

### 4.4 Employee Status Management

**Status Types:**
- **Active:** Currently employed
- **Probation:** Under probationary period
- **On Leave:** Currently on approved leave
- **Suspended:** Temporarily suspended
- **Terminated:** Employment ended

To change status:
1. Open employee profile
2. Click **Change Status**
3. Select new status
4. Provide reason if required
5. Confirm the change

## 6. Payroll Management

### 6.1 Payroll Overview

The payroll module handles:
- Salary computation
- Tax calculations (PAYE)
- SSNIT contributions
- Allowances and deductions
- Bank file generation
- Payslip generation

### 6.2 Payroll Periods

Payroll runs within defined periods:
- **Status Types:**
  - OPEN: Ready for processing
  - PROCESSING: Currently being computed
  - COMPUTED: Calculations complete
  - APPROVED: Ready for payment
  - PAID: Payment completed
  - CLOSED: Period finalized

### 6.3 Processing Payroll

1. Navigate to **Administration** → **Process Payroll**
2. Select the payroll period
3. Review employees included
4. Click **Compute Payroll**
5. Review computation results
6. Click **Approve** to finalize
7. Generate bank files and payslips

### 6.4 Employee Transactions

Add one-time or recurring transactions:

1. Go to **Administration** → **Employee Transactions**
2. Click **Add Transaction**
3. Select:
   - Employee
   - Transaction Type (Allowance/Deduction)
   - Amount
   - Effective Period
4. Click **Save**

**Transaction Types:**
- Overtime
- Bonus
- Arrears
- Loan Deduction
- Advance Deduction
- Other Allowances/Deductions

### 6.5 Viewing Payslips

**For Employees:**
1. Go to **My Profile** → **Payslips**
2. Select the period
3. View or download payslip

**For HR:**
1. Go to **Payroll** → Select period
2. Click **Payslips**
3. Download individual or bulk payslips

### 6.6 Bank File Generation

1. Navigate to processed payroll run
2. Click **Generate Bank Files**
3. Files are created per bank
4. Download CSV/Excel files for bank upload

## 7. Performance & Appraisals

### 7.1 Appraisal Cycle

The appraisal process follows this flow:

1. **Goal Setting** (January)
   - Employee sets goals
   - Manager approves goals

2. **Mid-Year Review** (July)
   - Progress review
   - Goal adjustments

3. **Year-End Appraisal** (December)
   - Self-assessment
   - Manager review
   - Final rating

### 7.2 Creating an Appraisal (HR/Admin)

1. Navigate to **Administration** → **Appraisals**
2. Click **New Appraisal**
3. Select:
   - Appraisal Cycle
   - Employee
4. Click **Create Appraisal**

### 7.3 Setting Goals (Employee)

1. Go to **My Appraisal**
2. Click **Add Goal**
3. Enter:
   - Goal Title
   - Description
   - Success Criteria
   - Target Date
   - Weight (percentage)
4. Click **Save**
5. Submit goals for manager approval

### 7.4 Self-Assessment (Employee)

1. Go to **My Appraisal**
2. For each goal:
   - Rate your achievement (1-5)
   - Add comments
3. Complete competency ratings
4. Complete core values assessment
5. Click **Submit Self-Assessment**

### 7.5 Manager Review

1. Navigate to **My Appraisal** → **Team Appraisals**
2. Select employee appraisal
3. Review self-assessment
4. Provide manager ratings
5. Add comments and feedback
6. Click **Complete Review**

### 7.6 Appraisal Scoring

Scores are weighted as follows (configurable):
- **Objectives/Goals:** 60%
- **Competencies:** 20%
- **Core Values:** 20%

**Rating Scale:**
| Rating | Description | Score Range |
|--------|-------------|-------------|
| 5 | Outstanding | 90-100% |
| 4 | Exceeds Expectations | 75-89% |
| 3 | Meets Expectations | 60-74% |
| 2 | Needs Improvement | 40-59% |
| 1 | Unsatisfactory | 0-39% |

### 7.7 Probation Assessments

For employees on probation:

1. Go to **Administration** → **Probation Assessments**
2. View employees due for assessment
3. Complete assessment form:
   - Job Knowledge
   - Work Quality
   - Attendance & Punctuality
   - Teamwork
   - Communication
   - Initiative
4. Make recommendation:
   - Confirm employment
   - Extend probation
   - Terminate

## 11. Company Policies

### 11.1 Overview

The Company Policy module allows HR to create, publish, and track staff acknowledgement of organizational policies, SOPs, guidelines, and circulars.

### 11.2 Policy Categories

Policies are organized into categories:
- Human Resources
- Finance & Accounting
- Information Technology
- Operations
- Compliance & Legal
- Health & Safety
- Administration

### 11.3 Creating a Policy

1. Navigate to **Administration** → **Company Policies**
2. Click **New Policy**
3. Fill in the required fields:
   - **Code**: Unique identifier (e.g., HR-POL-001)
   - **Title**: Policy title
   - **Category**: Select from available categories
   - **Type**: Policy, SOP, Guideline, Manual, Circular, or Memo
   - **Summary**: Brief description
   - **Content**: Full policy text (Markdown supported)
4. Set version and dates:
   - **Version**: Policy version number
   - **Effective Date**: When policy comes into effect
   - **Review Date**: Next scheduled review
5. Configure acknowledgement settings:
   - **Requires Acknowledgement**: Staff must acknowledge reading
   - **Deadline (days)**: Days to acknowledge after publication
6. Click **Create Policy**

### 11.4 Publishing Policies

1. Navigate to the policy detail page
2. Review the content
3. Click **Publish**
4. Staff will be notified and required to acknowledge

### 11.5 Staff Acknowledgement

**For Employees:**
1. Go to **Company Policies** (or receive notification)
2. Read the policy content
3. Click **Acknowledge**
4. Optionally add comments
5. Submit acknowledgement

**For HR (Tracking):**
1. Navigate to policy detail page
2. View **Acknowledgements** tab
3. See who has acknowledged and when
4. Monitor pending acknowledgements

### 11.6 Version Control

When a published policy is updated:
- Previous version is automatically saved
- Version history is maintained
- Staff can view previous versions

## 12. Exit Management / Offboarding

### 12.1 Overview

The Exit Management module handles the complete employee offboarding process including:
- Resignation processing
- Retirement handling
- Termination workflows
- Exit interviews
- Department clearances
- Asset returns
- Final settlement calculation

### 12.2 Exit Types

Common exit types include:
- **Resignation** - Voluntary departure (30 days notice)
- **Retirement** - End of service (90 days notice)
- **Termination** - Employment ended by organization
- **End of Contract** - Contract period ended
- **Redundancy** - Position made redundant
- **Dismissal** - Dismissal for misconduct
- **Transfer Out** - Transfer to another organization

### 12.3 Creating an Exit Request

1. Navigate to **Administration** → **Exit Management**
2. Click **New Exit Request**
3. Fill in the form:
   - Select Employee
   - Select Exit Type
   - Enter Proposed Last Working Day
   - Provide Reason for Exit
   - Add any Additional Comments
4. Click **Create Request**

### 12.4 Exit Request Workflow

**Status Flow:**
1. **Draft** - Request created but not submitted
2. **Submitted** - Awaiting approval
3. **Approved** - Request approved, clearance begins
4. **Clearance** - Awaiting department clearances
5. **In Progress** - All clearances complete, final steps
6. **Completed** - Exit process finalized

### 12.5 Department Clearances

Each departing employee must obtain clearance from designated departments:

**Standard Clearance Departments:**
- Human Resources (ID card, personnel file, exit interview)
- Information Technology (laptop, email access, system accounts)
- Finance (loans, advances, expense claims)
- Administration (keys, parking, equipment)
- Direct Supervisor (work handover, knowledge transfer)
- Department Head (departmental clearance)
- Security (access badges, biometrics)

**Completing a Clearance:**
1. Navigate to the exit request
2. Go to **Clearances** tab
3. Click **Clear** on your department
4. Complete checklist items
5. Add any comments or amounts owed
6. Submit clearance

### 12.6 Asset Returns

Track company assets that need to be returned:

1. From the exit request, go to **Assets** tab
2. Click **Add Asset** to record items
3. Enter asset details (name, type, tag number)
4. Process each return:
   - Mark as Returned
   - Mark as Damaged (with deduction)
   - Mark as Lost/Missing (with deduction)
   - Mark as Purchased by Employee

### 12.7 Exit Interview

Conduct exit interviews to gather feedback:

1. Navigate to the exit request
2. Go to **Exit Interview** tab
3. Schedule the interview
4. Record feedback:
   - Reason for leaving
   - Job satisfaction ratings
   - What they liked most/least
   - Suggestions for improvement
   - Would they recommend the organization
5. Mark interview as completed

### 12.8 Final Settlement

Calculate and process the final settlement:

**Earnings:**
- Salary arrears
- Leave encashment
- Gratuity
- Bonus
- Other earnings

**Deductions:**
- Outstanding loan balance
- Advance balance
- Asset deductions
- Tax deductions
- Other deductions

**Process:**
1. Go to **Final Settlement** tab
2. Enter earnings and deductions
3. Click **Calculate** to compute totals
4. Submit for approval
5. Once approved, process payment
6. Mark as paid with payment reference

### 12.9 Completing an Exit

Once all steps are complete:
1. All clearances must be completed
2. Final settlement must be approved
3. Click **Complete Exit**
4. Employee status is automatically updated

## Appendix

### A. Glossary

| Term | Definition |
|------|------------|
| **PAYE** | Pay As You Earn - Income tax deducted at source |
| **SSNIT** | Social Security and National Insurance Trust |
| **TIN** | Tax Identification Number |
| **Gross Pay** | Total earnings before deductions |
| **Net Pay** | Take-home pay after all deductions |
| **Appraisal Cycle** | Annual performance review period |
| **Probation** | Trial period for new employees |

### B. Status Codes

**Employee Status:**
- ACTIVE - Currently employed
- PROBATION - Under probation
- ON_LEAVE - On approved leave
- SUSPENDED - Temporarily suspended
- TERMINATED - Employment ended

**Leave Status:**
- PENDING - Awaiting approval
- APPROVED - Leave granted
- REJECTED - Leave denied
- CANCELLED - Leave cancelled

**Exit Request Status:**
- DRAFT - Request created, not yet submitted
- SUBMITTED - Awaiting approval
- PENDING_APPROVAL - Under review
- APPROVED - Request approved
- REJECTED - Request rejected
- CLEARANCE - Awaiting department clearances
- IN_PROGRESS - Clearances complete, final steps
- COMPLETED - Exit process complete
- CANCELLED - Request cancelled
- WITHDRAWN - Request withdrawn

**Payroll Status:**
- OPEN - Ready for processing
- PROCESSING - Being computed
- COMPUTED - Calculation complete
- APPROVED - Ready for payment
- PAID - Payment done
- CLOSED - Period closed
